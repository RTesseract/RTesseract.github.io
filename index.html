Research

  • Reconciling HTM and NVM Programming with Buffered Durability [WIP]
    Supervisors: Prof. Michael Scott, Ph.D. candidate Mingzhe Du
    Adopted and internalized a concurrent vEB (Van Emde Boas) tree using Hardware Transactional Memory (HTM) for synchronization. Enhanced the code to provide persistence across program runs and crashes, maintaining the correctness requirements of buffered durable linearizability. Based persistence on the Montage library, which forces updates flush to persistent memory on a periodic basis. For up to 20 threads, the performance of the persistent vEB tree using HTM is 9 times that of using a global lock. (Beyond that point, hardware transactions frequently abort, due to cross-thread sharing.)

  • Persistent-memory calculator (GNU bc) project [WIP]
    Supervisor: Dr. Terence Kelly
    Forked and modified an existing GNU/Linux calculator utility called bc. Redirected user-defined data structures in bc from the volatile to non-volatile memory and enabled access to data structures in future invocations without any data loss, leading to less code in the scenario of incremental data update. Successfully developed a prototype and performed extensive testing. Updated in response to changes in maintainers' demands. Currently negotiating with maintainers for merging into new bc releases. Solidifies experiences debugging with GDB.

  • Improving U-Net Performance for Low-Grade Glioma MRI Image Segmentation by Modification of U-Net Concatenations [link]

Become more familiar with TensorFlow syntax and third-party platforms such as Colab and TensorBoard.
